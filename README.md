# Minecraft Seed Map JP v5.2

Minecraftのシード値から、疑似バイオーム、スライムチャンク、構造物候補、手動マーカーを確認できる日本語の静的Seed Mapです。

v5.2では、構造物候補の出し方、探索範囲、Canvas上のアイコン視認性を改善しました。

## v5.2の更新内容

- 構造物ごとに `spacing` / `separation` / `salt` を持つ候補生成へ整理
- seed、region座標、構造物種別から決定的に候補座標を生成
- Java版/統合版で別パラメータを利用
- 疑似バイオーム条件で候補を評価
- 候補ごとに `confidence` を追加
  - 高
  - 中
  - 低
- 右側詳細に「候補精度」「候補理由」「対象バージョン」を表示
- 表示範囲より少し広い範囲まで候補を生成
- 近隣候補が途切れにくいように構造物候補をキャッシュ
- Canvas上の構造物アイコンを種類別に描画
  - 村: 家風
  - 廃ポータル: 門風
  - 海底神殿: 神殿風
  - 森の洋館: 屋敷風
  - ピリジャー前哨基地: 旗風
  - 古代都市: 遺跡風
  - トライアルチャンバー: 銅色の四角
  - 要塞: 砦/目風
- ホバーした構造物アイコンを強調表示
- 構造物候補数とキャッシュ範囲数をUIに表示

## 候補精度について

構造物候補はMinecraft本体の完全な生成ロジックではありません。現在は、seed、region座標、構造物タイプ、疑似バイオームから推定した候補です。

例:

- 村: 草原、砂漠、サバンナ、雪原を高精度候補として扱う
- 海底神殿: 海を高精度候補として扱う
- 森の洋館: 暗い森を高精度候補として扱う
- 古代都市: 山岳/雪原系を高精度候補として扱う
- 廃ポータル/トライアルチャンバー: 広域候補として扱う

将来的には、cubiomes WASMなど正確な構造物生成providerへ差し替える予定です。

## ローカル実行

ES Modulesを使っているため、プロジェクトルートで静的サーバーを起動してください。

```powershell
python -m http.server 8000
```

ブラウザで以下を開きます。

```text
http://localhost:8000/
```

## 動作確認ポイント

- Java版で構造物候補が広い範囲に表示される
- Bedrock版で構造物候補が広い範囲に表示される
- seedを変えると候補位置が変わる
- 同じseedでは候補位置が再現される
- ズームアウトしても構造物候補が見える
- ドラッグ移動しても近隣候補が途切れにくい
- アイコンが構造物ごとに判別しやすい
- ホバー/クリックで詳細が分かる
- スライムチャンクON/OFFと独立して構造物候補が表示される
- 通常操作でコンソールエラーや頻繁な `console.warn` が出ない

## 保存と通信

- 外部通信なしで動作します
- ユーザー入力はブラウザ内の `localStorage` 以外に保存しません
- GitHub Pagesで公開できる静的構成です

## 注意

このツールはChunkbaseの完全な代替ではありません。v5.2時点では、構造物とバイオームは候補表示・疑似表示として扱っています。
