# Minecraft Seed Map JP v2.0

Minecraftのシード値から、スライムチャンクを日本語UIで確認する静的Webツールです。v2.0はスライムチャンク表示とは別に、手動登録した構造物を専用レイヤーとしてマップ上に表示できます。外部API、外部ライブラリ、アクセス解析は使用していません。

## 主な機能

- 数値または文字列のシード値入力
- Java版のスライムチャンク判定（対応済み）
- 統合版のスライムチャンク判定（実験的対応、今後検証が必要）
- Minecraftバージョン選択
- 中心X/Z座標と表示範囲の指定
- チャンク単位のスライムチャンクグリッド表示
- 選択チャンクのチャンク座標、ブロック範囲、中心ブロック座標、判定表示
- チャンク座標、中心ブロック座標、ブロック範囲の個別コピー
- オーバーワールド座標とネザー座標の相互変換
- 地点メモの登録、一覧表示、削除、全削除
- 表示中マップ範囲内の手動登録構造物を専用レイヤーで表示
- 構造物レイヤーのON/OFF切り替え
- 村、要塞、廃ポータルなどの構造物カテゴリ管理
- カテゴリ別のマーカー色分け
- チェックボックスによるカテゴリフィルタ
- 地点名検索とカテゴリ検索
- 指定座標へのジャンプ、原点へ戻る、地点一覧からの移動
- 現在の中心チャンク座標と中心ブロック座標の表示
- スマートフォン幅に対応したレスポンシブ表示

## セットアップ

追加インストールは不要です。リポジトリを取得し、`index.html` をブラウザで開くだけで動作します。

```text
minecraft-seed-map-jp/
├── index.html
├── README.md
├── css/
│   └── style.css
├── js/
│   ├── main.js
│   ├── slime.js
│   ├── storage.js
│   └── utils.js
└── assets/
    └── icons/
        └── favicon.svg
```

## ローカル実行

このプロジェクトはビルド不要です。基本的には `index.html` をブラウザで開くだけで確認できます。

ブラウザのセキュリティ制限でJavaScriptモジュールが読み込めない場合は、プロジェクトのルートディレクトリで静的サーバーを起動してください。

```powershell
python -m http.server 8000
```

その後、ブラウザで `http://localhost:8000/` を開きます。

## 動作確認ポイント

ローカルで開いたら、以下を確認してください。

1. `index.html` を開くと、条件入力フォーム、チャンクグリッド表示エリア、地点メモ欄が表示される。
2. シード値、中心X/Z、表示範囲を入力して `マップを生成` を押すと、チャンクグリッドが生成される。
3. グリッド上のチャンクをクリックすると、チャンク座標、ブロック範囲、中心ブロック座標、スライム判定が表示される。
4. `チャンク座標をコピー`、`中心座標をコピー`、`ブロック範囲をコピー` で、必要な座標情報を個別にコピーできる。
5. ネザー座標変換で、オーバーワールド座標とネザー座標を整数のX/Zで相互変換できる。
6. 地点名、X/Z座標、カテゴリ、メモを登録すると一覧に表示される。
7. 登録地点が現在表示中のマップ範囲内にある場合、構造物レイヤー上のマーカーとして表示される。
8. カテゴリごとにマーカーや一覧の色が変わる。
9. 構造物レイヤーのON/OFFを切り替えられる。
10. チェックボックスで全表示またはカテゴリ別表示を切り替えられる。
11. 地点名検索とカテゴリ検索で一覧とマーカーを絞り込める。
12. マーカー付きチャンクをクリックすると、チャンク詳細内に構造物の詳細が表示される。
13. 中心X/Zを入力して `この座標へ移動` を押すと、指定座標周辺のマップに再描画される。
14. `原点へ戻る` を押すと、X=0 / Z=0 を中心に再描画される。
15. 地点一覧の `移動` を押すと、その地点周辺のマップに移動できる。
16. マップ上に現在の中心チャンク座標と中心ブロック座標が表示される。
17. ページを再読み込みしても、地点メモがブラウザの `localStorage` に残っている。
18. 画面幅を狭くしても、フォームやグリッド周辺の表示が大きく崩れない。

## GitHub Pagesでの公開

1. GitHubに `minecraft-seed-map-jp` リポジトリを作成します。
2. このディレクトリの内容を `main` ブランチへ push します。
3. GitHubの `Settings` → `Pages` を開きます。
4. `Build and deployment` を `Deploy from a branch` にします。
5. Branchを `main`、Folderを `/root` にして保存します。

## 使い方

1. シード値を入力します。
2. エディション、バージョン、中心座標、表示範囲を選びます。
3. `マップを生成` を押します。
4. グリッド上のチャンクを選択すると、チャンク座標とブロック範囲が表示されます。
5. 必要に応じて座標コピー、ネザー座標変換、地点メモを使います。
6. チャンク詳細の `この座標を地点登録に使う` で、選択チャンクの中心座標を地点登録フォームへ入れられます。
7. 中心X/Zを変更して `この座標へ移動`、または地点一覧の `移動` で表示範囲を切り替えます。

## 入力項目

- シード値: 数値または文字列。文字列はJavaの `String.hashCode` 相当で数値化します。
- 中心X/Z: ブロック座標の整数値。
- 表示範囲: 16、32、64チャンクから選択します。

## 出力内容

- チャンクグリッド: スライムチャンクは緑、通常チャンクはグレー、中心付近は枠で強調します。
- 構造物レイヤー: 手動登録した構造物をスライムチャンクとは別レイヤーとして表示します。
- 選択チャンク詳細: チャンク座標、ブロック座標範囲、中心ブロック座標、スライム判定。
- ネザー座標変換: オーバーワールド座標とネザー座標のX/Z変換結果。小数が出る場合は四捨五入して表示します。
- 地点メモ: 地点名、X/Z座標、カテゴリ、メモ本文。
- マーカー: 現在表示中のマップ範囲内にある地点メモをチャンク上に表示します。
- 構造物カテゴリ: 村、要塞、廃ポータル、海底神殿、森の洋館、ピリジャー前哨基地、古代都市、トライアルチャンバー、ネザー要塞、砦の遺跡、エンドシティ、その他。
- フィルタと検索: カテゴリチェックボックス、地点名検索、カテゴリ検索で地点メモとマーカーを絞り込めます。
- 表示中心: 現在の中心チャンク座標と中心ブロック座標を表示します。

## セキュリティとプライバシー

このツールは外部通信を行いません。入力されたシード値、座標、地点名、カテゴリ、メモ本文は外部サーバーへ送信されません。

地点メモはブラウザの `localStorage` にのみ保存されます。共有PCや公開端末で使う場合は、使用後に画面内の `全削除` を押すか、ブラウザのサイトデータを削除してください。

## 注意事項

- このツールは非公式のMinecraft補助ツールです。
- MinecraftおよびMojang Studiosとは関係ありません。
- Java版のスライムチャンク判定は対応済みです。
- 統合版のスライムチャンク判定はv2.0では実験的対応です。判定結果は今後検証が必要です。
- v2.0の構造物表示は手動登録のみ対応です。構造物の自動検出は今後対応予定です。
- バイオーム、地形、3D表示、ログイン、外部API連携、広告、アクセス解析、PWAには対応していません。
