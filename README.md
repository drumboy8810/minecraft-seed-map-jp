# Minecraft Seed Map JP v5.0

Minecraftのシード値から、疑似バイオーム、スライムチャンク、構造物候補、手動マーカーを確認できる日本語の静的Seed Mapです。

v5.0では、従来のHTMLグリッド描画からCanvasベースのマップエンジンへ移行しました。Chunkbase Seed Mapのような操作感に近づけるため、ドラッグ移動、マウスホイールズーム、疑似バイオームの細かい描画、構造物アイコンの重ね表示を追加しています。

- マウスホイールによるズームに対応

  - 標準
  - 詳細
- 疑似バイオーム/地形表示を強化
  - 海
  - 川
  - 草原
  - 森
  - 暗い森
  - 砂漠
  - サバンナ
  - ジャングル風
  - 雪原
  - 山岳
  - 湿地
  - 荒野風
- Canvas上に以下のレイヤーを重ねて表示
  - 疑似バイオーム
  - 座標グリッド
  - スライムチャンク
  - 構造物候補アイコン
  - 手動マーカー
- Java版/統合版の両方で構造物候補を表示
- スライムチャンクON/OFFに依存せず構造物候補を表示
- 将来のcubiomes WASM差し替えに備え、provider構成を追加

以下の構造物を候補として表示します。

- 村
- 要塞
- 廃ポータル
- 海底神殿
- 森の洋館
- ピリジャー前哨基地

現在の構造物表示は候補表示です。実際のMinecraft生成位置やChunkbaseの表示と完全一致する保証はありません。バイオーム成立判定、生成失敗条件、正確なバージョン差分は今後の課題です。

## provider構成

v5.0では、将来の正確な生成ロジックに差し替えやすいように、描画と判定を分離しています。

- `js/map/map-engine.js`
  - Canvas描画
  - pan/zoom
  - レイヤー合成
- `js/map/biome-provider.js`
  - 疑似バイオーム取得
  - 将来の詳細バイオーム差し替え入口
- `js/map/structure-provider.js`
  - 表示範囲内の構造物候補取得
- `js/map/slime-provider.js`
  - スライムチャンク判定入口
- `js/terrain/simple-terrain-provider.js`
  - 現在利用している疑似バイオームprovider
- `js/biome/cubiomes-provider.js`
  - cubiomes WASM導入用の準備入口

## cubiomes WASMについて

正確なJava版バイオーム判定や構造物生成に近づけるには、将来的にcubiomesのWebAssembly統合が必要です。

現時点では以下の状態です。

- cubiomes本体は同梱していません
- `assets/wasm/` は将来のWASM配置先です
- 詳細バイオームモードは準備中です
- WASM未配置でもアプリは壊れず、疑似バイオームモードで動作します

## ローカル実行

このプロジェクトは静的HTML/CSS/JavaScriptのみで動作します。ES Modulesを使っているため、ブラウザで直接 `index.html` を開いて動かない場合は、プロジェクトルートで静的サーバーを起動してください。

```powershell
python -m http.server 8000
```

その後、ブラウザで以下を開きます。

```text
http://localhost:8000/
```

## 動作確認ポイント

- シードを入力してマップ生成できる
- マップをドラッグで移動できる
- マウスホイールでズームできる
- seedを変えると疑似バイオーム表示が変わる
- Java版/統合版の両方で構造物候補が表示される
- スライムチャンクOFFでも構造物候補が表示される
- 構造物カテゴリフィルタで表示が切り替わる
- マップクリックで右側の詳細が更新される
- 座標コピーが動く
- ネザー座標変換が動く
- 地点メモがlocalStorageに保存され、手動マーカーとして表示される
- 通常操作でコンソールエラーや頻繁な `console.warn` が出ない

## 保存と通信

- 外部通信なしで動作します
- ユーザー入力はブラウザ内の `localStorage` 以外に保存しません
- GitHub Pagesで公開できる静的構成です

## 注意

このツールはChunkbaseの完全な代替ではありません。v5.0時点では、操作感と見た目をSeed Mapらしく近づけることを優先し、構造物とバイオームは候補表示・疑似表示として扱っています。
